## Описание

Создана простая система управления задачами с использованием Spring Boot и Spring Data JPA.
Система позволяет создавать, просматривать, обновлять и удалять задачи.

## Установка и запуск

1. Склонируйте репозиторий: `git clone https://github.com/AntonAshutov/taskManager.git`
2. Убедитесь, что на вашем компьютере установлен Maven
3. Откройте проект в вашей IDE
4. Убедитесь что порт 8080 вашего компьютера свободен или установите другой в application.yaml
5. Запустите проект

## Технологии

- H2

- REST API
- JUnit

## API Endpoints

API для управления задачами.

### Получить список всех задач

```
GET /api/v1/tasks/
```

Возвращает список всех задач.

#### Пример ответа

```json
[
  {
    "id": 1,
    "title": "Название задачи",
    "description": "Описание задачи",
    "dueDate": "2024-04-19T12:00:00",
    "completed": false
  },
  {
    "id": 2,
    "title": "Другая задача",
    "description": "Описание другой задачи",
    "dueDate": "2024-04-20T12:00:00",
    "completed": true
  }
]
```

### Получить информацию о задаче по ID

```
GET /api/v1/tasks/{id}
```

Возвращает информацию о задаче с указанным ID.

#### Параметры пути

| Параметр | Тип    | Описание                                                                  |
|----------|--------|---------------------------------------------------------------------------|
| `id`     | `long` | **Обязательный**. Номер задачи, информацию по которой вы хотите получить. |

#### Пример ответа

```json
{
  "id": 1,
  "title": "Название задачи",
  "description": "Описание задачи",
  "dueDate": "2024-04-19T12:00:00",
  "completed": false
}
```

### Создать новую задачу

```
POST /api/v1/tasks/
```

Создает новую задачу с указанными данными.

Тело запроса

```json
{
  "id": null,
  "title": "Название новой задачи",
  "description": "Описание новой задачи",
  "dueDate": "2024-04-21T12:00:00",
  "completed": false
}
```

id указывать нет смысла - он проигнорируется и выставится базой

#### Ответ

- `200 OK` при успешном запросе

### Обновить информацию о задаче

```
PUT /api/v1/tasks/{id}
```

Обновляет информацию о задаче с указанным ID.

#### Параметры пути

| Параметр | Тип    | Описание                                                                  |
|----------|--------|---------------------------------------------------------------------------|
| `id`     | `long` | **Обязательный**. Номер задачи, информацию по которой вы хотите обновить. |

Тело запроса

```json
{
  "id": null,
  "title": "Новое название задачи",
  "description": "Новое описание задачи",
  "dueDate": "2024-04-22T12:00:00",
  "completed": true
}
```
Опять же, в теле id указывать смысла нет, его изменить не получится
Пример ответа

- `200 OK` при успешном запросе

### Удалить задачу

```
DELETE /api/v1/tasks/{id}
```

Удаляет задачу с указанным ID.

#### Параметры пути

| Параметр | Тип    | Описание                                                   |
|----------|--------|------------------------------------------------------------|
| `id`     | `long` | **Обязательный**. Номер задачи, которую вы хотите удалить. |

- `200 OK` при успешном запросе
